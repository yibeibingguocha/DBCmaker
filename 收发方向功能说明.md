# DBC转化工具 - 收发方向功能使用说明

## 新增功能概述

本次更新增加了**收发方向识别**功能，可以正确区分不同CAN路之间同一报文的收发方向。

### 问题背景
- ADASCAN的0x580报文是发出（T）
- BCAN的0x580报文是接收（R）
- 原代码无法区分，将所有报文都错误地添加为发送

### 解决方案
在Excel信号矩阵中增加一列"收发方向"，标记每个报文的传输方向（T=发送，R=接收）。

---

## 使用方法

### 1. Excel文件格式要求

在您的Excel信号矩阵中添加一列，列名可以是以下任意一种：
- `收发方向`
- `方向`
- `direction`
- `TX/RX`
- `T/R`
- `收发`
- `传输方向`

该列的值应为：
- **T** 或包含T：表示发送（Transmit）
- **R** 或包含R（但不含T）：表示接收（Receive）

示例：
```
| 报文名称 | 报文ID | 信号名称 | ... | 收发方向 |
|---------|--------|---------|-----|---------|
| Msg1    | 0x580  | Sig1    | ... | T       |
| Msg2    | 0x580  | Sig2    | ... | R       |
```

### 2. 工具操作步骤

1. **运行工具**：双击运行 `DBC转化工具v1.0.0.0.py`

2. **选择Excel文件**：点击"选择XLSX文件输入目录"按钮

3. **自动识别列**：工具会自动检测列名并填充到对应输入框
   - 如果检测到"收发方向"相关列，会自动填充
   - 也可以手动输入列名或列号（如：I 或 9）

4. **配置收发方向列**（可选）：
   - 在"输入收发方向所在列(可选)"输入框中确认或修改
   - 如果不设置，默认所有报文为发送(T)

5. **生成DBC文件**：点击"生成DBC文件"按钮

### 3. 输出结果

生成的DBC文件中：
- **发送报文（T）**：正常添加到对应节点的BO_定义中
- **接收报文（R）**：也会添加到BO_定义中，但会添加注释标记 `"接收报文(R)"`

示例DBC输出：
```dbc
BO_ 1408 Msg1: 8 ADASCAN
 SG_ Sig1 : 0|8@1+ (1,0) [0|0] "" Vector__XXX

BO_ 1408 Msg2: 8 BCAN
 SG_ Sig2 : 0|8@1+ (1,0) [0|0] "" Vector__XXX
CM_ BO_ 1408 "接收报文(R)";
```

## 技术说明

### DBC格式限制
DBC文件的BO_（报文）定义只能指定一个发送节点。接收关系通常通过以下方式表示：
1. BO_TX_BU_ 属性（本工具暂未实现）
2. 注释说明（本工具采用此方式）
3. CANoe等工具的额外配置

### 节点管理
- 每个Sheet对应一个节点（使用Sheet名作为节点名）
- TBOX节点会被特殊识别并单独列出
- 所有节点都会在BU_行中声明

### 智能匹配规则
工具会自动识别包含以下关键词的列名：
- 收发方向、方向、direction、tx/rx、t/r、收发、传输方向

匹配优先级：
1. 精确匹配（100分）
2. 包含匹配（50分）
3. 被包含匹配（30分）

## 常见问题

### Q1: 如果不设置收发方向列会怎样？
A: 所有报文默认为发送(T)，保持原有行为。

### Q2: 收发方向列的值可以是中文吗？
A: 可以，只要包含"T"或"R"字符即可识别。例如："发送"、"接收"、"Tx"、"Rx"。

### Q3: 同一个ID在不同Sheet中有不同方向怎么办？
A: 这正是本功能要解决的问题。每个Sheet独立处理，根据各自的收发方向列正确标识。

### Q4: TBOX关键词的作用是什么？
A: 如果某列的表头包含"TBOX"且单元格值为"TX"，则该报文归属于TBOX节点。
